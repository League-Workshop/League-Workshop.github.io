Pre-Arrival

The workshop gives the first impression of The League to most students. Therefore, before the students arrive, check the classroom as follows:

  • Check there are enough computers for expected student attendance.
  • Clear computer Desktop folders of unwanted content.
  • Make sure there is no active GitHub session.
  • Check Eclipse is empty of projects.
  • Wipe down keyboards and mice. Clean screens.
  • Print waivers for students who do not have a signed waiver on file.
  • For multi-class workshops, organize students in age groups and post student names outside classrooms.

Survey / Github

Ask students to complete the pre-workshop survey: bit.ly/LeagueBeforeWorkshop

  • Use this activity to establish if all students have set up a Github account

Attendance and Photos

Sign into your (teacher) Pike13 account.

  • Mark attendance for each student
  • Take a head shot photo and upload it into each student's client profile

Warm-up

This web site provides activities for students to work on while they wait for everyone to arrive and log in. bit.ly/blockly-maze. Be sure to emphasize that this is NOT Java coding.

Clone Workshop Code

Students will need to get the workshop code from GitHub. See below:

Students without GitHub accounts can work on a copy of the workshop code (you will need to download this for them) but explain that their work will not be saved at the end of the class.

  • Sign in to GitHub
  • Go to: bit.ly/introtojava
  • Allow GitHub classroom to access the GitHub account.
  • Accept the assignment from the classroom.
  • Check they have a copy of the LeagueWorkshop/intro-to-java-workshop-XXXXXXX repository (where XXXXXXX is their github id)
  • Clone the code to the computer's Desktop.
  • NOTE: There is no need to import this code into Eclipse at this time. The first session uses Processing.
